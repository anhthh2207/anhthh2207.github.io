Configurable Logic Block Design, Verification and Optimization

In this project, I with my teammate, Linh Chu, designed a 16-bit Configurable Logic Block (CLB), a circuit that can model any 4-input combinational logic function. It is an essential block that provides flexibility for Field Programmable Gate Arrays (FPGAs). The circuit was developed in 45nm Salicide 1.0V/1.8V 1P 11M technology supported in Cadence.

Key Components of the Configurable Logic Block includes:

  • 16:1 Lookup Table (LUT)
  • SRAM Array
  • Serial In Parallel Out (SIPO) Register
  • Non-overlapping Clock Generator

Finally, these components are integrated to form a comprehensive CLB design:

  • Configurable Logic Block

16:1 Lookup Table (LUT)

The first component in our design is the 16:1 LUT. It accepts 4 binary inputs and stores 16 output values, corresponding to 24 possible input combinations. When the LUT receives 4 input values, these values are used as an address to look up the corresponding output from the pre-stored array. The beauty of LUT is its configurability; by changing the pre-stored array, the LUT can implement any logical function.

We begin with the design of a 2:1 MUX (similar to LUT, it selects one of two outputs based on a select line). A LUT can be constructed by connecting 15 MUXes in a tree structure, as shown below. Two CMOS logic inverters at the output function as serve as a buffer, restoring the voltage after pass-transistor logic degradation in the MUXes.

SRAM Array

SRAM cells are used to store the possible outputs for the LUT. The figure below shows the design of a single 6T SRAM cell, whose read-write are controlled by the Bit lines (BL and BL) and the Word line (WL) signals through precharge and discharge actions. See a nice tutorial about how an SRAM cell works.

However, to control the memory cell more robustly and facilitate integration with other components, we have designed additional peripherals and ports:

  • PC: Precharge condition
  • W_EN: Write enable
  • WL: Word line
  • DATA: Write input
  • OUT: Read output

Sixteen identical cells are combined into a 16-bit SRAM array.

16-bit SRAM array

Serial In Parallel Out (SIPO) Register

To load data into the SRAM array, we use a SIPO register. It converts serial data into parallel data by receiving one bit at a time and then outputting the entire set of taken bits simultaneously as a parallel word.

A SIPO register is constructed by cascading 16 D flip-flops, as demonstrated below.

Non-overlapping Clock Generator

As can be observed, several components, particularly the flip-flops, utilize both CLK and CLK as inputs. Consequently, we developed a module to generate two complementary clock signals from a single source.

Configurable Logic Block (CLB)

All modules are integrated into a unified CLB design supporting any 4-input combinational function.

Performance Metrics

After benchmarking our design, the performance parameters of the circuit are reported in the table below:

MetricValue
Max frequency1 GHz
Loading Energy3.212 × 10-3 nJ
Active Energy1.895 × 10-3 nJ

* Loading Energy: energy consumed by the entire circuit to load all the data into the SRAM

* Active Energy: energy consumed to cycle through all possible input combinations from 0 to 15 at the maximum operating frequency
